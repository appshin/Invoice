# KOWOO 인보이스 QR 라벨 발행

Packing list(인보이스) 엑셀 파일을 올리면 **(8) No. & Date of invoice** 항목의
인보이스 번호와 날짜를 자동으로 읽어 **QR 라벨을 원하는 매수만큼 인쇄**하는 웹앱입니다.

서버 없이 브라우저에서만 동작하므로 GitHub Pages에 그대로 올려서 쓸 수 있습니다.
**엑셀 파일은 어디에도 업로드되지 않고 브라우저 안에서만 읽습니다.**

## 화면 구성

| 파일 | 용도 |
|------|------|
| `index.html` | **라벨 발행** — 엑셀 업로드 → 파렛트별 QR 라벨 인쇄 |
| `scan.html` | **중복 스캔 검사** — 카메라/스캐너로 읽어 중복 알람 |

## 사용 흐름

1. 엑셀 파일(.xlsx)을 드래그하거나 클릭해서 선택
2. 인보이스 번호·날짜·품목·파렛트 구성이 자동으로 채워짐
3. 품목 체크박스로 포함/제외, 인쇄 범위 지정
4. 미리보기 확인 → **인쇄**
5. 현장에서 `scan.html` 로 각 파렛트 라벨을 스캔 → 중복 시 경고

### 라벨 수 계산 방식

`(10) Description of Goods` 의 파렛트 구성에서 자동 계산합니다.

| 품목 | 구성 | 라벨 |
|------|------|------|
| 1. RB C-MDPS 32.0x2.8 | 1800 × 4 + 900 × 1 | 5 |
| 2. NX4/KA4 MSCL | 1200 × 15 | 15 |
| | | **20** |

`ONE PALLET` / `PARTIAL PALLET` / `PARTIAL PALLET` 3개 그룹의
`No. of pallet` 를 모두 더한 값이 총 라벨 수입니다.

## 중복 스캔 검사 (`scan.html`)

- **카메라 스캔** — 휴대폰·태블릿으로 라벨을 비추면 자동 인식 (https 필요)
- **현장 스캐너** — USB·블루투스 스캐너는 입력칸에 그대로 인식
- 이미 스캔한 라벨을 다시 읽으면 **빨간 화면 + 경고음 + 진동**으로 알림
- 진행 현황(12/20), 남은 번호, 스캔 기록, CSV 내보내기
- 기록은 브라우저에 저장되어 새로고침해도 유지됩니다

## QR 내용

스캔하면 아래 내용이 그대로 읽힙니다.

```
INVOICE: KW260716-RM
DATE: Jul-16-2026
ITEM: 1. COLD DRAWN PIPE(RB C-MDPS)_SIZE : 32.0x2.8
QTY: 1,800 PCS
NO: 03/20          ← 라벨마다 다른 구분값
```

`NO` 덕분에 QR 내용이 라벨마다 전부 달라 중복 스캔으로 잡히지 않습니다.

**재인쇄(라벨 손상 교체) 시**: 인쇄 범위의 시작·끝을 그 번호로 맞추면
**원래 라벨과 완전히 같은 QR**이 나옵니다. 같은 파렛트를 가리키는 라벨이므로
새 번호가 생기지 않아야 재고가 이중으로 잡히지 않습니다.

## 인보이스 자동 인식 방식

1. 시트에서 **"No. & Date of invoice"** 라벨 문구를 찾아 **바로 아래 행**에서
   인보이스 번호(문자열)와 날짜(날짜 셀)를 읽습니다.
2. 못 찾으면 고정 위치 **G4 / J4** 를 확인합니다.
3. 그래도 못 찾으면 화면에 안내가 뜨고 직접 입력할 수 있습니다.

양식이 조금 바뀌어도 라벨 문구만 있으면 계속 동작합니다.
날짜는 엑셀 서식(`mmm-d-yyyy`)과 동일하게 `Jul-16-2026` 형태로 표시됩니다.

## 라벨 형식

```
┌──────────────────────────────────────────────┐
│  ┌────────┐   INVOICE NO.                    │
│  │        │   ┏━━━━━━━━━━━━━━━┓        NO.   │
│  │   QR   │   ┃  KW260716-RM  ┃         03   │
│  │        │   ┗━━━━━━━━━━━━━━━┛        / 20  │
│  │        │   DATE                           │
│  └────────┘   ┏━━━━━━━━━━━━━━━┓              │
│               ┃  Jul-16-2026  ┃              │
│               ┗━━━━━━━━━━━━━━━┛              │
└──────────────────────────────────────────────┘
     QR          노란 하이라이트 = 육안 식별용
```

- **스캔이 안 되는 상황에서도** 인보이스 번호·날짜를 눈으로 바로 읽을 수 있습니다
- 오른쪽 끝 **NO. 03/20** 으로 라벨끼리 구분됩니다
- A4 세로 한 장에 **2개** (라벨 194 × 136mm · QR 70mm)
- QR은 **벡터(SVG)** 라 축소 인쇄해도 스캔이 잘 됩니다
- 노란 하이라이트가 배경색이라 인쇄 옵션에서 "배경 그래픽"이 꺼져 있으면
  색만 빠질 수 있으나, **QR·글자·괘선은 그대로 인쇄**되어 식별에 지장 없습니다

## 커스터마이징

- QR 내용 형식: `assets/js/app.js` 의 `payload()` (바꾸면 `scan.js` 의 `parsePayload()` 도 함께)
- 라벨 디자인/크기: `assets/js/app.js` 의 `labelHtml()`, `assets/css/app.css`
- 추출 규칙: `assets/js/parse.js` 의 `extract()`
